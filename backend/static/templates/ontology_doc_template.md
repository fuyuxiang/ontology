## 二、业务流程

描述端到端业务流程，明确每个环节的角色、动作和状态流转。

### 流程描述格式

```
步骤序号. [角色] 执行动作 → 产生结果/状态变更
```

### 状态流转

列出核心业务对象的状态变化路径：

```
状态A → 触发条件 → 状态B → 触发条件 → 状态C
```

**示例：**

### 宽带装机工单流程

1. [客户] 通过营业厅/APP提交宽带装机申请 → 生成正式工单
2. [调度系统] 自动派单给片区工程师 → 工单状态变为"已派单"
3. [工程师] 上门施工或反馈退单 → 工单状态变为"已完工"或"退单"
4. [稽核系统] 对退单工单进行原因核实 → 判定是否为虚假退单
5. [管理人员] 根据稽核结果进行考核处置 → 更新考核记录

### 工单状态流转

```
待派单 → 自动派单 → 已派单 → 工程师接单 → 施工中 → 完工/退单
退单 → 稽核中 → 稽核通过/稽核驳回
```

## 填写建议

1. **第三章是核心**：对象和属性描述得越完整，AI 抽取越精准
2. **关系要写清基数**：1:1、1:N、N:N 会直接影响本体结构
3. **规则要有具体判定逻辑**：不要只写"判断是否异常"，要写清用什么条件判断
4. **数据源字段对应**：属性能对应到具体的 表.字段 会大幅提升映射准确度
5. **没有的章节可以留空**：AI 会在对话中追问缺失的关键信息
