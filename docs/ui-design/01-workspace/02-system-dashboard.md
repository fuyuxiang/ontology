# 系统看板（System Dashboard）

> 工作台 · 面向运维和管理员的系统健康监控页面

---

## 1. 页面概述

| 维度 | 说明 |
|------|------|
| 页面定位 | 平台运维监控中心，实时展示系统健康状态与关键性能指标 |
| 目标用户 | 系统管理员、运维工程师、技术负责人 |
| 核心价值 | 快速发现系统异常，支撑故障定位与容量规划决策 |
| 路由路径 | `/workspace/system-dashboard` |
| 所属分组 | 工作台 |

---

## 2. 功能清单

| # | 功能点 | 优先级 | 交互类型 | 说明 |
|---|--------|--------|----------|------|

## 4. 各区域详细设计

### 4.1 页面标题与刷新控制栏

| 属性 | 值 |
|------|-----|
| 布局 | Flex，`justify-content: space-between`，`align-items: center` |
| 左侧标题 | "系统看板" - `--text-h1` 22px / 600 / `--neutral-900` |
| 右侧信息 | "最后更新: HH:mm:ss" - 14px / 400 / `--neutral-500` |
| 刷新按钮 | `a-dropdown` 下拉选择刷新频率：30s / 60s / 5min / 关闭 |
| 手动刷新 | ReloadOutlined 图标按钮，点击立即刷新所有数据 |
| 间距 | 底部 margin-bottom: 20px |

### 4.2 服务健康状态概览

| 属性 | 值 |
|------|-----|
| 组件 | `a-card` 包裹，内部为横向 Flex 布局 |
| 卡片高度 | 80px |
| 内边距 | 16px 20px |
| 服务项数量 | 6 个（可扩展） |
| 单项布局 | 状态圆点 + 服务名 + 响应时间，垂直居中 |

**单个服务项**：

```
  [●] 后端 API
      12ms
```

| 属性 | 值 |
|------|-----|
| 状态圆点 | 直径 10px，健康 #12b886 / 警告 #f59f00 / 异常 #fa5252 |
| 圆点动画 | 异常时脉冲闪烁动画 (`pulse`)，周期 1.5s |
| 服务名称 | `--text-body` 14px / 500 / `--neutral-700` |
| 响应时间 | `--text-caption` 12px / 400 / `--neutral-500` |
| 各项间距 | gap: 32px，均匀分布 |
| hover | 整个服务项背景变为 `--neutral-50`，圆角 6px |
| 点击 | 展开该服务的详细健康信息面板（抽屉或弹窗） |

**状态判定规则**：

| 状态 | 条件 | 颜色 | 标签 |
|------|------|------|------|
| 健康 | 响应时间 <500ms 且可用率 >99.5% | #12b886 | 正常 |
| 警告 | 响应时间 500-2000ms 或可用率 95-99.5% | #f59f00 | 注意 |
| 异常 | 响应时间 >2000ms 或可用率 <95% 或不可达 | #fa5252 | 异常 |

### 4.3 资源使用率仪表盘

| 属性 | 值 |
|------|-----|
| 组件 | `a-card` 包裹，内部 3 个环形图并排 |
| 卡片标题 | "资源使用率" - `--text-h3` 15px / 600 |
| 卡片高度 | 320px |
| 内边距 | 20px |
| 布局 | 3 列等宽 Flex，gap: 24px |

**单个环形图**：

| 属性 | 值 |
|------|-----|
| 图表组件 | ECharts gauge 或 `a-progress` type="circle" |
| 环形直径 | 120px |
| 环形粗细 | 12px |
| 中心数值 | 28px / 700 / `--neutral-900` |
| 下方标签 | 资源名 14px / 500 + 具体数值 12px / 400 |
| 颜色规则 | 0-60%: `--dynamic-500` #20c997 / 60-85%: `--kinetic-500` #f59f00 / 85-100%: #fa5252 |
| 动画 | 数值变化时平滑过渡，300ms ease |

**三个指标**：

| 指标 | 标签 | 附加信息 |
|------|------|----------|
| CPU | CPU 使用率 | 核心数、当前频率 |
| 内存 | 内存使用率 | 已用/总量（如 12.8G / 32G） |
| 磁盘 | 磁盘使用率 | 已用/总量（如 256G / 1T） |

### 4.4 服务响应时间折线图

| 属性 | 值 |
|------|-----|
| 组件 | `a-card` 包裹 ECharts 折线图 |

## 5. 数据展示规范

### 5.1 数字格式化

| 场景 | 规则 | 示例 |
|------|------|------|
| 百分比 | 整数或1位小数 | 73% / 85.2% |
| 响应时间 | <1000 显示 ms，>=1000 转为 s | 128ms / 2.3s |
| 内存/磁盘 | 自动选择合适单位 | 12.8G / 256G |
| 调用量 | 千位逗号，超万用万单位 | 8,432 / 12.5万 |
| 时间戳 | HH:mm:ss 格式 | 10:32:15 |

### 5.2 空状态

| 区域 | 空状态展示 |
|------|-----------|
| 服务健康 | 所有服务显示灰色圆点 + "检测中..." |
| 资源仪表盘 | 环形图显示 0%，灰色描边 |
| 响应时间图 | 空坐标轴 + "暂无监控数据" |
| 告警列表 | 绿色对勾图标 + "当前无告警，系统运行正常" |
| 事件流 | "暂无系统事件" |

### 5.3 加载态

| 区域 | 加载方式 |
|------|----------|
| 服务健康 | 骨架屏圆点 + 横条 |
| 仪表盘 | `a-skeleton` 圆形占位 |
| 图表 | `a-spin` 居中 |
| 告警列表 | `a-skeleton` 表格行骨架，5 行 |
| 事件流 | `a-skeleton` 列表骨架，5 行 |

### 5.4 错误态

| 场景 | 处理方式 |
|------|----------|
| 监控接口不可用 | 对应区域显示 "监控服务连接失败" + 重试按钮 |
| 部分服务不可达 | 该服务项显示灰色 + "不可达" 标签 |
| WebSocket 断连 | 顶部通知条 "实时连接已断开，正在重连..." |

### 5.5 数据刷新策略

| 区域 | 刷新方式 | 默认频率 |
|------|----------|----------|
| 服务健康状态 | 轮询 | 30s |
| 资源使用率 | 轮询 | 30s |
| 响应时间图 | 轮询追加数据点 | 60s |
| 告警列表 | WebSocket 推送 | 实时 |
| 事件流 | WebSocket 推送 | 实时 |

## 6. 响应式适配

| 断点 | 服务健康栏 | 中间区域 | 底部区域 | 页面 padding |
|------|-----------|----------|----------|-------------|
| >=1920px | 单行6项 | 左40% + 右60% | 左50% + 右50% | 32px |
| 1440-1919px | 单行6项 | 左40% + 右60% | 左50% + 右50% | 24px |
| 1280-1439px | 单行6项（紧凑） | 左40% + 右60% | 左50% + 右50% | 20px |
| <1280px | 两行各3项 | 上下堆叠 | 上下堆叠 | 16px |

**具体规则**：

- 服务健康栏 <1280px 时分两行展示，每行 3 个服务
- 资源仪表盘 <1280px 时环形图缩小至 90px 直径
- 响应时间图 <1280px 时高度缩至 220px
- 告警列表和事件流 <1280px 时上下堆叠，各高度 280px
- 表格在窄屏时隐藏"服务"列，告警内容列允许换行
