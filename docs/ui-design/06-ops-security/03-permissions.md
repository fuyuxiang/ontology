# 权限管理（Permissions）

> 运维与安全中心 · RBAC 用户/角色/权限三元管理

---

## 1. 页面概述

| 维度 | 说明 |
|------|------|
| 页面定位 | 基于 RBAC 模型的统一权限管理中心，覆盖用户、角色、权限矩阵三层管理 |
| 目标用户 | 系统管理员、安全管理员 |
| 核心价值 | 精细化控制平台访问权限，确保最小权限原则，满足企业安全合规要求 |
| 路由路径 | `/ops/permissions` |
| 所属分组 | 运维与安全中心 |

---

## 4. 各区域详细设计

### 4.1 页面标题区

| 属性 | 规格 |
|------|------|
| 标题 | `text-display` 28px / 700 / `--neutral-900` |
| 副标题 | `text-caption` 12px / 400 / `--neutral-500`，margin-top: 4px |
| 间距 | 底部 margin-bottom: 16px |

### 4.2 Segmented 切换

| 属性 | 规格 |
|------|------|
| 组件 | `a-segmented`（Ant Design Vue） |
| 选项 | "用户管理" / "角色管理" / "权限矩阵" |
| 尺寸 | 默认尺寸 |

### 4.3 用户管理工具栏

| 属性 | 规格 |
|------|------|
| 容器 | flex / gap: 12px / align-items: center |
| 搜索框 | `a-input-search`，width: 280px，placeholder "搜索姓名或账号" |
| 角色筛选 | `a-select`，width: 160px，placeholder "全部角色"，选项为系统角色列表 |
| 状态筛选 | `a-select`，width: 120px，选项: 全部/启用/禁用 |
| 新增按钮 | Primary 类型，图标 PlusOutlined + "新增用户" |
| 新增按钮位置 | margin-left: auto（右对齐） |

### 4.4 用户表格

| 属性 | 规格 |
|------|------|
| 组件 | 自定义 table（与项目现有风格一致） |
| 表头 | 背景 `--neutral-50` / 12px / 600 / `--neutral-500` |
| 行高 | 48px |
| 姓名列 | width: 120px / 14px / 600 / `--neutral-900` |
| 账号列 | width: 160px / 13px / font-mono / `--neutral-700` |
| 角色列 | width: 140px / 标签样式（与角色卡片颜色呼应） |
| 角色标签配色 | 系统管理员: T1色 / 本体工程师: T2色 / 数据工程师: T3色 / 业务分析师: `--kinetic-500` / 只读用户: `--neutral-500` |
| 状态列 | width: 80px / 圆点+文字 / 启用: 绿色 / 禁用: 灰色 |
| 最近登录列 | width: 140px / 13px / font-mono / `--neutral-500` |
| 操作列 | width: 80px / 更多按钮(EllipsisOutlined) + 下拉菜单(编辑/禁用/重置密码) |
| 分页 | 底部右对齐 / 显示总数 / 每页条数可选(10/20/50) |

### 4.5 新增/编辑用户模态框

| 属性 | 规格 |
|------|------|
| 组件 | `a-modal`，width: 520px |
| 标题 | "新增用户" / "编辑用户" / 18px / 600 |
| 表单布局 | 垂直布局(vertical)，label 在上方 |
| 字段列表 | 账号(必填)、姓名(必填)、邮箱(必填，邮箱验证)、手机号(选填)、角色(必填，Select多选)、初始密码(新增时必填) |

| 角色选择 | `a-select` mode="multiple"，标签样式展示已选角色 |
| 底部按钮 | 右对齐: [取消](Ghost) + [确定](Primary) |
| 验证 | 实时校验 + 提交时统一校验 / 账号需检查唯一性(防抖请求) |

| 角色名称 | 16px / 600 / `--neutral-900` / margin-top: 12px |

| 底部统计 | 13px / `--neutral-500` / "N人 · M项权限" |

| 删除操作 | 卡片右上角更多菜单(仅自定义角色可见) |

### 4.7 角色编辑抽屉

| 属性 | 规格 |
|------|------|
| 组件 | `a-drawer`，width: 640px |
| 头部 | 角色名称 + 描述编辑区 |
| 权限树 | `a-tree` 组件，checkable，分模块分组 |
| 树结构层级 | 第一层: 功能模块（本体中心、数据集成...） / 第二层: 子功能（如: 本体浏览、本体编辑、本体发布） / 第三层: 操作级别（读取、写入、管理） |
| 全选/取消 | 每个模块标题行有全选复选框 |
| 半选态 | 模块部分子项选中时显示半选(indeterminate) |
| 底部操作栏 | 固定定位 / border-top: 1px solid `--neutral-200` / padding: 12px 24px / [取消] + [保存](Primary) |
| 系统角色限制 | 系统预置角色: 权限树只读展示(disabled)，底部提示 "系统预置角色不可修改" |

### 4.8 权限矩阵

| 属性 | 规格 |
|------|------|
| 容器 | 水平可滚动(overflow-x: auto)，固定首列(sticky) |
| 表头 | 背景 `--neutral-50` / 12px / 600 / 文字居中 |
| 首列(角色名) | width: 140px / 14px / 500 / 背景 `--neutral-0` / position: sticky left: 0 |
| 单元格 | width: 100px / 居中 / padding: 8px |
| 权限级别展示 | ■管理: 实心方块 `--semantic-500` / ◧写: 半填充 `--dynamic-500` / □读: 空心 `--kinetic-500` / ─: 横线 `--neutral-300` |
| 单元格交互 | 点击循环切换: 无权限 → 读 → 写 → 管理 → 无权限 |
| 单元格 hover | 背景高亮 `--semantic-50` |
| 图例 | 表格下方，flex布局，13px / `--neutral-600` |
| 保存 | 矩阵修改后右上角出现 "保存变更" Primary 按钮(sticky) |
| 变更标记 | 修改过的单元格加虚线边框标识未保存状态 |

### 4.9 预定义角色说明

| 角色 | 描述 | 核心权限 |
|------|------|----------|
| 系统管理员 | 平台最高权限，管理所有模块和用户 | 所有模块: 管理 |
| 本体工程师 | 负责本体建模和场景开发 | 本体中心: 管理 / 场景中心: 管理 / 其他: 读 |
| 数据工程师 | 负责数据集成和质量治理 | 数据集成: 管理 / 治理: 管理 / 其他: 读 |
| 业务分析师 | 使用场景和Agent进行业务分析 | 场景中心: 写 / Agent: 写 / 其他: 读 |
| 只读用户 | 仅浏览平台内容，不可修改 | 所有模块: 读 |

## 5. 数据展示规范

| 场景 | 规范 |
|------|------|
| 用户总数 | 显示为 "共 N 人" |
| 最近登录时间 | 格式 "MM-DD HH:mm"，超过7天显示完整日期 |
| 从未登录 | 显示 "从未登录" / `--neutral-400` / 斜体 |
| 角色标签 | 多角色用逗号分隔，超过2个显示 "+N" |
| 权限数量 | 整数，如 "28项权限" |
| 加载态 | 用户表格: 行级骨架屏 / 角色卡片: 卡片级骨架屏(3个) / 矩阵: 网格骨架 |
| 空状态(用户) | 图标: 用户组 / 标题: "暂无用户" / 描述: "点击新增按钮添加第一个用户" / 按钮: "新增用户" |
| 空状态(角色) | 图标: 盾牌 / 标题: "暂无自定义角色" / 描述: "系统已预置5个标准角色" |
| 错误态 | Alert 组件红色提示 + "重试"按钮 |
| 删除确认 | 危险模态框: "确定要删除角色「XX」吗？此操作不可撤销。" |
| 关联检查 | 删除角色时如有关联用户，禁用删除按钮 + 提示 "该角色下有N个用户，请先移除用户" |

## 6. 响应式适配

| 断点 | 布局变化 |
|------|----------|
| Desktop L (≥1920px) | 角色卡片3列 / 矩阵全列可见 / 页面 padding: 32px |
| Desktop M (1440-1919px) | 角色卡片3列 / 矩阵横向滚动 / 页面 padding: 24px |
| Desktop S (1280-1439px) | 角色卡片2列 / 用户表格隐藏邮箱列 / 页面 padding: 20px |
| Tablet (<1280px) | 角色卡片1列 / 用户表格仅显示核心列(姓名/角色/状态/操作) / 抽屉改为全屏 / 页面 padding: 16px |

权限矩阵响应式：
- ≥1920px：所有列平铺显示
- 1440-1919px：首列 sticky + 横向滚动
- <1440px：切换为列表模式（每个角色一个卡片，内部列出各模块权限）
