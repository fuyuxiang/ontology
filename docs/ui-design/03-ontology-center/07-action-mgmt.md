# 行动管理

> 本体中心 · 由规则触发的动作定义与执行管理

---

## 1. 页面概述

| 维度 | 说明 |
|------|------|
| 页面定位 | 管理本体中由规则触发或手动执行的动作（Action），包括API调用、消息推送、数据写入等多种类型，提供注册、配置和执行监控 |
| 用户角色 | 本体工程师、系统集成工程师、运维工程师 |
| 核心价值 | 将规则的"动作端"独立管理，实现动作的可复用、可监控、可独立测试，降低规则与执行逻辑的耦合 |

---

| F04 | 删除动作 | P1 | 确认弹窗 | 删除动作定义 |

| F06 | 分类筛选 | P1 | Tag过滤 | 按状态/触发方式筛选（全部/已激活/手动/自动） |

| F08 | 执行历史 | P2 | 折叠面板 | 查看动作的历史执行记录 |

| F10 | 关联规则查看 | P2 | 链接跳转 | 查看哪些规则引用了该动作 |

## 4. 各区域详细设计

### 4.1 标题栏

| 属性 | 值 |
|------|-----|
| 布局 | flex，两端对齐 |
| 标题 | `--text-display` 28px / 700 / `--neutral-900` |
| 副标题 | `--text-caption` 12px / 400 / `--neutral-500`，margin-top 4px |
| 新建按钮 | Primary Button，带 + 图标，"新建动作" |

### 4.2 统计卡片区

| 属性 | 值 |
|------|-----|
| 布局 | grid 4列，gap 16px |
| 卡片 | 高 72px，圆角 8px，内边距 16px |
| 背景 | `--neutral-0` |
| 数值 | `--text-display` 28px / 700 |
| 标签 | `--text-caption` 12px / 400 / `--neutral-500` |

左边框颜色：
- 总动作数：3px solid `--semantic-500`
- 已激活：3px solid `--dynamic-500`
- 手动触发：3px solid `--kinetic-500`
- 自动触发：3px solid `--status-error`

### 4.3 筛选栏

| 属性 | 值 |
|------|-----|
| 布局 | flex，gap 12px，align-items center |
| 搜索框 | 宽 280px，高 36px，圆角 4px |
| 筛选Tag | pill 按钮组，与规则管理一致 |
| Tag选项 | 全部 / 已激活 / 手动 / 自动&API |

### 4.4 动作卡片

| 属性 | 值 |
|------|-----|
| 组件 | 自定义 ActionCard（复用 rule-card 样式） |
| 容器 | 圆角 8px，border 1px solid `--neutral-200`，margin-bottom 8px |
| 点击 | 切换展开/收起 |
| hover | 阴影 `--shadow-sm` |
| 展开态 | border-color `--semantic-500` |

卡片头部（收起态）：

| 元素 | 规格 |
|------|------|
| 状态圆点 | 8px，active=`--dynamic-500` / inactive=`--neutral-300` / warning=`--kinetic-500` |
| 动作名 | 14px / 600 / `--neutral-900` |
| 关联实体 | 12px / pill / `--neutral-500` 底色 `--neutral-100` |
| 触发类型 | pill 标签，manual=蓝 / auto=绿 / api=紫 |
| 行高 | 48px，padding 0 16px |

卡片展开详情：

| 元素 | 规格 |
|------|------|
| 容器 | padding 16px，上边框 1px dashed `--neutral-200` |
| 触发方式行 | label(80px) + 内容 |
| 执行次数行 | label + 数字 + "次" |
| 参数行 | label + code 字体显示参数名列表 |
| 操作按钮 | 执行(橙色 `--kinetic-500`) + 删除(红色) |

触发类型标签颜色：
- manual：背景 `--semantic-50`，文字 `--semantic-600`
- auto：背景 `#e6fcf5`，文字 `--dynamic-500`
- api：背景 `#f3f0ff`，文字 `#7950f2`

### 4.5 新建动作弹窗

| 属性 | 值 |
|------|-----|
| 组件 | `a-modal`（项目中的 ModalDialog），width 520px |
| 标题 | "新建动作" |
| 表单字段 | 动作名称(必填)、关联实体(下拉必填)、触发方式(下拉:手动/自动/API) |
| 底部 | "取消" + "创建"（Primary） |

表单字段规格：

| 字段 | 组件 | 规格 |
|------|------|------|
| 动作名称 | `a-input` | required，placeholder "如：发送告警通知" |
| 关联实体 | `a-select` | 从实体列表中选择，显示中文名(英文名) |
| 触发方式 | `a-select` | 选项：手动/自动/API |

### 4.6 执行反馈

| 场景 | 表现 |
|------|------|
| 执行中 | 按钮变为 loading 态 + "执行中..." |
| 执行成功 | 全局 Toast 成功提示 + 执行次数 +1 |
| 执行失败 | 全局 Toast 错误提示 + 错误信息 |

## 5. 数据展示规范

| 场景 | 处理方式 |
|------|----------|
| 数字格式 | 执行次数整数显示，千分位分隔（如 1,234 次） |
| 空状态 | "无匹配动作"居中显示，搜索无结果时提供清除搜索按钮 |
| 加载态 | 统计卡片 Skeleton + 列表 3 个卡片骨架 |
| 错误态 | API失败时 message.error 全局提示 |
| 参数显示 | JSON 参数以逗号分隔的 code 样式展示参数名 |
