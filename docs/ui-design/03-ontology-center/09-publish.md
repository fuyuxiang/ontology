# 本体发布

> 本体中心 · 版本化管理与多阶段发布流程

---

## 1. 页面概述

| 维度 | 说明 |
|------|------|
| 页面定位 | 管理本体的版本化发布流程，包括版本创建、实体选择、一致性校验、审批提交和正式发布，确保发布质量可控 |
| 用户角色 | 本体工程师（创建/提交）、审核管理员（审批/驳回）、运维工程师（发布/回滚） |
| 核心价值 | 引入版本管理和审批流程，避免未经验证的变更进入生产环境，支持版本回滚保障系统稳定性 |

---

## 4. 各区域详细设计

### 4.1 标题区

| 属性 | 值 |
|------|-----|
| 标题 | `--text-h1` 22px / 600 / `--neutral-900` |
| 描述 | 14px / 400 / `--neutral-500` |
| 内边距 | 24px 32px |

### 4.2 左栏 - 版本时间线

| 属性 | 值 |
|------|-----|
| 宽度 | 280px |
| 背景 | `--neutral-0` |
| 右边框 | 1px solid `--neutral-200` |
| 内边距 | 16px |
| 溢出 | overflow-y: auto |

创建按钮：

| 属性 | 值 |
|------|-----|
| 类型 | Primary，宽度 100% |
| 文字 | "创建新版本" |
| margin-bottom | 16px |

版本项：

| 属性 | 值 |
|------|-----|
| 高度 | auto（约 56px） |
| 内边距 | 10px 12px |
| 圆角 | 8px |
| hover | 背景 `--neutral-50` |
| 选中态 | 背景 `--semantic-50`，左边框 3px solid `--semantic-500` |
| margin-bottom | 4px |

版本项内部：

| 元素 | 规格 |
|------|------|
| 版本号 badge | pill 12px / 700，颜色根据状态变化 |
| 版本名称 | 13px / 500 / `--neutral-900` |
| 元信息 | 12px / `--neutral-500`，"3 个实体 · 草稿" |
| 当前标记 | 如为活跃版本，右侧显示 "当前" 绿色pill |

版本状态颜色：
- draft（草稿）：背景 `--neutral-100`，文字 `--neutral-600`
- submitted（已提交）：背景 `--kinetic-50`，文字 `--kinetic-600`
- approved（已批准）：背景 `--semantic-50`，文字 `--semantic-600`
- published（已发布）：背景 `#e6fcf5`，文字 `--dynamic-500`
- rejected（已驳回）：背景 `--status-error-bg`，文字 `--status-error`

### 4.3 右栏 - 版本元信息卡片

| 属性 | 值 |
|------|-----|
| 圆角 | `--radius-lg` 12px |
| 背景 | `--neutral-0` |
| 边框 | 1px solid `--neutral-200` |
| 内边距 | 20px |
| margin-bottom | 16px |

内部元素：

| 元素 | 规格 |
|------|------|
| 标题行 | "v2.1 — 用户域扩展" 18px/600 + 状态 Tag 右侧 |
| 描述 | 14px / 400 / `--neutral-600`，margin-top 8px |
| 时间线 | 12px / `--neutral-500`，多个时间节点水平排列，圆点+文字 |
| 驳回原因 | 如有，红色背景条，12px 红色文字 |
| 回滚来源 | 如有，蓝色提示条 |

### 4.4 右栏 - 实体列表卡片

| 属性 | 值 |
|------|-----|
| 卡片样式 | 同元信息卡片 |
| 头部 | "包含实体 (3)" 16px/600 + "添加实体" 按钮（草稿时可见） |

实体表格：

| 列 | 宽度 | 规格 |
|----|------|------|
| 实体名称 | 35% | 中文名(13px/600) + 英文名(11px/`--neutral-400`) |
| 层级 | 10% | "T1" / "T2" / "T3" |
| 属性映射 | 25% | "8/12" 格式，表示已映射/总属性 |
| 就绪状态 | 15% | ✓ 绿色 / ✗ 红色 |
| 操作 | 15% | "移除" 链接（草稿时可见） |

### 4.5 右栏 - 一致性检查卡片

| 属性 | 值 |
|------|-----|
| 显示条件 | 执行过检查后显示 |
| 通过态 | 绿色背景条 `#e6fcf5`，"✓ 所有检查项通过" |
| 失败态 | 红色背景条 `#fff5f5`，摘要文字 + 问题列表 |

问题列表项：

| 属性 | 值 |
|------|-----|
| 样式 | 每行 padding 8px，12px 文字，实体名加粗 |
| 问题类型 | 属性问题/关系问题，分区展示 |
| 图标 | 问题前缀 ⚠ 或 ✗ |

### 4.6 操作栏

| 属性 | 值 |
|------|-----|
| 布局 | flex，gap 12px |
| 位置 | 版本详情底部 |
| 按钮根据版本状态动态显示 |

状态到按钮映射：

| 状态 | 可见按钮 |
|------|----------|
| draft | [一致性检查](Secondary) + [提交审批](Primary) + [删除草稿](Danger Outline) |
| submitted | [审批通过](Primary) + [驳回](Danger Outline) |
| approved | [正式发布](Primary) |
| published | [回滚到此版本](Secondary) |

## 5. 数据展示规范

| 场景 | 处理方式 |
|------|----------|
| 数字格式 | 版本号格式 "v1.0" / "v2.1"；实体数直接整数 |
| 时间格式 | "创建: 2024-01-15 14:30" |
| 空状态 - 无版本 | 左栏显示"暂无版本" + 创建按钮高亮 |
| 空状态 - 无实体 | 实体区显示"暂未添加实体，点击上方按钮选择" |
| 加载态 | 左栏版本列表 Skeleton + 右栏卡片 Skeleton |
| 错误态 | 操作失败时全局 Toast + 按钮恢复可用状态 |
| 按钮禁用 | 无实体时"提交审批"禁用 + tooltip 提示原因 |
