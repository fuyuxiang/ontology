# 本体服务

> 本体中心 · API 接口管理 + OSDK 客户端生成

---

## 1. 页面概述

| 维度 | 说明 |
|------|------|
| 页面定位 | 本体服务化的统一出口，包含两大子模块：API 服务（接口浏览与在线调试）和 OSDK（多语言客户端SDK自动生成与下载） |
| 用户角色 | 后端开发工程师、前端开发工程师、数据接口消费者、API集成人员、SDK使用方 |
| 核心价值 | 将本体从静态模型转化为可调用的服务接口和开箱即用的客户端SDK，降低数据消费和集成门槛 |

## 5. 数据展示规范

| 场景 | 处理方式 |
|------|----------|
| 数字格式 | 端点数量直接整数；实体数量带括号 "(12)" |
| 空状态 - 无端点 | 全页居中："暂无API端点" + "请先发布本体" 链接 |
| 空状态 - 搜索无结果 | 左栏列表位置显示"未找到匹配端点" |
| 空状态 - OSDK未选实体 | 右侧代码预览区显示："请在左侧选择要生成SDK的实体" |
| 加载态 - API列表 | 左栏分组列表 Skeleton + 右栏 Skeleton |
| 加载态 - SDK生成 | 下载按钮 loading + 右侧代码区显示生成进度条 |
| 错误态 - 接口加载失败 | Toast 错误 + 左栏显示重试 |
| 错误态 - 调试请求失败 | 响应区显示错误信息（红色文字），HTTP状态码标注 |
| 错误态 - SDK生成失败 | Toast 错误 + 按钮恢复可点击状态 |
| 响应格式 | JSON 自动 pretty-print（2空格缩进） |
| 大响应 | 超过 300px 高度时 overflow scroll |
| SDK版本号 | 语义化版本 vX.Y.Z，跟随本体发布版本 |
