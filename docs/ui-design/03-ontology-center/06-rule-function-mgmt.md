# 规则与函数管理

> 本体中心 · 业务规则引擎与计算函数库管理

---

## 1. 页面概述

| 维度 | 说明 |
|------|------|
| 页面定位 | 管理本体的业务逻辑层（RBox），包括条件-动作规则和可复用计算函数，支持创建、编辑、测试和执行 |
| 用户角色 | 本体工程师、业务规则分析师、开发工程师 |
| 核心价值 | 将业务逻辑从代码中抽离为可管理的规则和函数，支持热更新和可视化管理，降低业务变更成本 |

---

## 4. 各区域详细设计

### 4.1 Tab 切换栏

| 属性 | 值 |
|------|-----|
| 高度 | 48px |
| 背景 | `--neutral-0` |
| 下边框 | 1px solid `--neutral-200` |
| Tab项 | 14px / 500，间距 32px，内边距 12px 0 |
| 选中态 | 文字 `--semantic-600` + 底部 2px solid `--semantic-500` |
| 默认态 | 文字 `--neutral-600` |

### 4.2 统计卡片区

| 属性 | 值 |
|------|-----|
| 布局 | grid 4列，gap 16px |
| 卡片尺寸 | 高 72px，圆角 `--radius-md` 8px |
| 内边距 | 16px |
| 数值 | `--text-display` 28px / 700 |
| 标签 | `--text-caption` 12px / 400 / `--neutral-500` |

卡片左边框颜色：
- 总规则数：`--semantic-500` 蓝色 3px 左边框
- 活跃规则：`--dynamic-500` 绿色
- 需关注：`--kinetic-500` 橙色
- 已禁用：`--status-error` 红色

### 4.3 筛选栏

| 属性 | 值 |
|------|-----|
| 高度 | 48px |
| 布局 | flex，gap 12px，align-items center |

### 4.5 函数Tab（结构类似规则）

函数卡片与规则卡片结构一致，差异点：

| 差异项 | 规则 | 函数 |
|--------|------|------|
| 统计卡片 | 总数/活跃/需关注/已禁用 | 总数/已激活/派生属性/独立函数 |
| 优先级标签 | high/medium/low | 派生属性/表达式/SQL/脚本 |
| 展开详情 | 条件+动作+触发次数 | 描述+返回类型+逻辑体+调用次数 |
| 操作按钮 | 执行/编辑/删除 | 测试/删除 |

## 5. 数据展示规范

| 场景 | 处理方式 |
|------|----------|
| 数字格式 | 触发次数/调用次数直接整数，大数千分位；时间使用 "YYYY-MM-DD HH:mm" |
| 空状态 | 无规则时："暂无业务规则" + "新建规则" 按钮；无函数时类似 |
| 加载态 | 统计卡片数字 Skeleton + 列表区 3个卡片骨架 |
| 错误态 | 执行/测试失败时 Toast 错误提示 + 错误详情 |
| 筛选无结果 | "无匹配规则" / "无匹配函数"，文字居中 |
