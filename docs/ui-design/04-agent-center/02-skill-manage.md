---

## 1. 页面概述

| 维度 | 说明 |
|------|------|
| 页面定位 | Agent 可调用的工具/能力单元管理中心，是 Agent 能力的原子化基础 |
| 用户角色 | 本体工程师、AI 应用开发者、系统管理员 |
| 核心价值 | 统一管理内置与自定义技能，让 Agent 编排时可快速选用并组合能力 |

技能(Skill)是 Agent 调用的最小能力单元，类似函数/工具。技能可以是平台内置的（本体查询、规则执行等），也可以是用户自定义的（API 端点封装、代码片段等）。

## 4. 各区域详细设计

### 4.1 页面标题区

| 属性 | 规格 |
|------|------|
| 标题 | "技能管理"，`--text-h1` 22px/600/`--neutral-900` |
| 描述 | "管理智能体可调用的工具与能力单元"，14px/400/`--neutral-500` |
| 新建按钮 | Primary Button，图标(+) + "新建技能" |
| 布局 | flex justify-between align-center |

### 4.2 类型标签栏

| 属性 | 规格 |
|------|------|
| 组件 | 自定义 Tab 切换（非 a-tabs，采用按钮组风格） |
| 标签项 | 高度 32px，padding 0 16px，圆角 16px(pill形) |
| 默认态 | 背景透明，文字 `--neutral-700`，14px/500 |
| 激活态 | 背景 `--semantic-50`，文字 `--semantic-500`，border 1px `--semantic-200` |
| 计数 | 12px/400/`--neutral-500`，紧跟标签文字后 |
| 间距 | 标签间 8px |
| 距工具栏 | margin-bottom 16px |

### 4.3 工具栏

| 组件 | 规格 |
|------|------|
| 搜索框 | `a-input-search`，宽度 300px，placeholder "搜索技能名称或描述..." |
| 分类筛选 | `a-select`，宽度 140px，选项: 全部分类/查询类/写入类/推理类/集成类 |
| 排序 | `a-select`，宽度 120px，选项: 默认/最常用/最新创建/名称 |
| 布局 | flex，搜索左对齐，筛选和排序右对齐 |
| 距卡片区 | margin-bottom 20px |

### 4.4 技能卡片

| 属性 | 规格 |
|------|------|
| 容器 | `grid-template-columns: repeat(auto-fill, minmax(240px, 1fr))`，gap 16px |
| 卡片尺寸 | 最小宽度 240px，高度自适应(约 160px) |
| 背景 | `#ffffff`，border 1px `--neutral-200`，圆角 8px |
| padding | 20px |
| hover | translateY(-2px) + shadow-md，过渡 200ms |
| 点击 | 打开详情抽屉 |

**卡片内部结构**：

```
+------------------------------------+
| [图标 36×36]   [类型标签]  [开关]  |  ← 顶部行
|                                    |
| 技能名称                           |  ← 15px/600
| 技能描述（2行省略）                 |  ← 13px/400/neutral-500
|                                    |
| [分类标签]  [引用数: 3个Agent]     |  ← 底部行
+------------------------------------+
```

| 子元素 | 规格 |
|--------|------|
| 图标 | 36×36px，圆角 8px，背景渐变(按分类着色) |
| 类型标签 | pill 形，12px：内置(蓝) / 自定义(紫) / 第三方(橙) |
| 开关 | `a-switch` size="small"，仅自定义技能显示 |
| 名称 | 15px/600/`--neutral-900`，单行省略 |
| 描述 | 13px/400/`--neutral-500`，line-clamp: 2 |
| 分类标签 | 12px，ghost 风格(无背景有边框)，如"查询类" |
| 引用数 | 12px/400/`--neutral-500`，如"3个Agent使用" |

**图标背景色方案(按功能分类)**：

| 分类 | 渐变色 | 图标 |
|------|--------|------|
| 查询类 | `linear-gradient(135deg, #339af0, #5c7cfa)` | 搜索/数据库 |
| 写入类 | `linear-gradient(135deg, #20c997, #38d9a9)` | 编辑/写入 |
| 推理类 | `linear-gradient(135deg, #7950f2, #9775fa)` | 大脑/逻辑 |
| 集成类 | `linear-gradient(135deg, #f59f00, #fcc419)` | 链接/API |

### 4.5 技能详情抽屉

| 属性 | 规格 |
|------|------|
| 宽度 | 640px |
| 组件 | `a-drawer` placement="right" |
| 头部 | 技能图标(24×24) + 名称(18px/600) + 类型标签 + 关闭按钮 |
| Tab | `a-tabs`，两个标签: "详情" / "测试" |

**详情 Tab 内容**：

| 区块 | 内容 |
|------|------|
| 基本信息 | 描述(正文)、分类、创建时间、更新时间、版本号 |
| 输入参数 | 表格: 参数名 / 类型 / 是否必填 / 描述 / 默认值 |
| 输出格式 | JSON Schema 代码块展示(等宽字体，背景 `--neutral-50`) |
| 实现方式 | API 端点(URL+Method) 或 代码片段(Monaco 只读) |

**测试 Tab 内容**：

| 区块 | 规格 |
|------|------|
| 参数表单 | 根据输入参数定义动态生成表单，各字段使用对应类型组件 |
| 执行按钮 | Primary Button "执行测试"，loading 态显示执行中 |
| 输出区域 | 背景 `--neutral-50`，圆角 8px，padding 16px |
| 输出格式 | JSON 高亮显示(等宽字体)，右上角复制按钮 |
| 执行信息 | 底部显示: 耗时(如 "230ms") + 状态(成功/失败) |

### 4.6 新建/编辑技能抽屉

| 属性 | 规格 |
|------|------|
| 宽度 | 640px |
| 组件 | `a-drawer` placement="right" |
| 头部 | "新建技能" 或 "编辑技能"，18px/600 |
| 底部操作栏 | 固定定位，border-top 1px，padding 12px 24px，[取消](Ghost) + [保存](Primary) |

**表单字段**：

| 字段 | 组件 | 校验规则 |
|------|------|----------|
| 技能名称 | `a-input`，max 50字符 | 必填，唯一性校验 |
| 描述 | `a-textarea`，4行，max 200字符 | 必填 |
| 分类 | `a-select`，选项同分类筛选 | 必填 |
| 图标 | 图标选择器(网格弹窗) | 可选，默认使用分类图标 |
| 输入参数 | 动态表格: 参数名+类型+必填+描述+默认值 | 至少0项 |
| 输出格式 | Monaco Editor (JSON Schema mode)，高度 160px | 必填 |
| 实现方式 | Radio: API端点 / 代码片段 | 必填 |
| API端点(条件显示) | URL(input) + Method(select: GET/POST/PUT/DELETE) + Headers(KV) | URL必填 |
| 代码片段(条件显示) | Monaco Editor (Python/JavaScript)，高度 240px | 必填 |

**输入参数动态表格**：

```
+----------+----------+------+------------------+----------+----+
| 参数名    | 类型      | 必填  | 描述              | 默认值    | 操作|
+----------+----------+------+------------------+----------+----+
| query    | string   | ✓    | 查询语句          | —        | [×]|
| limit    | integer  | —    | 返回数量限制       | 10       | [×]|
+----------+----------+------+------------------+----------+----+
| [+ 添加参数]                                                   |
+----------------------------------------------------------------+
```

类型选项: string / integer / number / boolean / array / object

## 5. 数据展示规范

### 5.1 数字格式化

| 数据类型 | 格式 | 示例 |
|----------|------|------|
| 创建时间 | YYYY-MM-DD HH:mm | 2024-06-01 14:30 |
| 执行耗时 | 智能单位 | 230ms / 1.2s / 3.5s |
| Agent引用数 | 整数 + 单位 | 3个Agent使用 |
| 版本号 | v + 主.次 | v1.2 |

### 5.2 空状态

| 场景 | 图标 | 标题 | 描述 | 操作 |
|------|------|------|------|------|
| 无技能 | 工具箱插图 120×120 | 暂无技能 | 创建自定义技能或浏览内置技能库 | [新建技能] |
| 搜索无结果 | 搜索图标 64×64 | 未找到匹配的技能 | 尝试更换关键词 | [清除搜索] |
| 测试无输出 | 空白文档 64×64 | 点击"执行测试"查看输出 | 填写输入参数后执行 | — |
| 测试失败 | 错误图标 48×48 | 执行失败 | 显示错误信息文本 | [重试] |

### 5.3 加载态

| 场景 | 方案 |
|------|------|
| 卡片列表加载 | 骨架屏，4×2 卡片骨架(矩形 + 2行文字) |
| 抽屉内容加载 | 抽屉打开后内部 Spin 居中 |
| 技能测试执行中 | 执行按钮 loading + 输出区域显示脉冲动画 |
| 保存中 | 底部保存按钮 loading 态 |

### 5.4 错误态

| 场景 | 方案 |
|------|------|
| 列表加载失败 | `a-result` status="error" + 重试按钮 |
| 保存失败 | `a-message.error` "保存失败: {原因}" |
| 测试执行错误 | 输出区域红色边框 + 错误信息(等宽字体) |
| 名称重复 | 输入框下方红色提示 "该名称已被使用" |
| 参数校验失败 | 表单字段红色高亮 + 错误提示文字 |

## 6. 响应式适配

| 断点 | 卡片列数 | 内容区 padding | 抽屉宽度 | 搜索框宽度 |
|------|----------|----------------|----------|------------|
| ≥1920px | 5列 | 32px | 640px | 320px |
| 1440-1919px | 4列 | 24px | 640px | 300px |
| 1280-1439px | 3列 | 20px | 560px | 260px |
| <1280px | 2列 | 16px | 100%全屏 | 200px |

**小屏适配策略**：
- <1280px 时，抽屉改为全屏模式
- 类型标签栏可横向滚动(overflow-x: auto)
- 工具栏改为两行布局(搜索一行，筛选+排序一行)
