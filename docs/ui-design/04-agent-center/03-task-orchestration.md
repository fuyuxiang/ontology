## 1. 页面概述

| 维度 | 说明 |
|------|------|
| 页面定位 | 定义 Agent 复杂多步骤任务流程的可视化编排与执行监控中心 |
| 用户角色 | 本体工程师、流程设计师、系统运维人员 |
| 核心价值 | 将多个技能/Agent 组合为可控的自动化工作流，支持定时触发与事件驱动 |

任务编排允许用户以流程图方式组装串行、并行、条件分支和循环逻辑，并提供完整的执行监控与调试能力。

## 5. 数据展示规范

### 5.1 数字格式化

| 数据类型 | 格式 | 示例 |
|----------|------|------|
| 执行时间 | YYYY-MM-DD HH:mm:ss | 2024-06-01 14:30:25 |
| 相对时间 | 智能切换 | 刚刚/5分钟前/3小时前/昨天 14:30 |
| 执行耗时 | 智能单位 | 230ms / 2.3s / 1m 12s / 5m 30s |
| 成功率 | 百分比 1位小数 | 98.5% |
| Cron 表达式 | 人类可读 + 原始值 | "每天 08:00" (0 8 * * *) |
| 迭代次数 | 整数 | 第 3/10 次 |

### 5.2 空状态

| 场景 | 图标 | 标题 | 描述 | 操作 |
|------|------|------|------|------|
| 无任务 | 流程图插图 120×120 | 暂无任务 | 创建第一个自动化任务流程 | [新建任务] |
| 搜索无结果 | 搜索图标 64×64 | 未找到匹配的任务 | 尝试更换关键词或清除筛选 | [清除筛选] |
| 无执行历史 | 时钟图标 64×64 | 暂无执行记录 | 运行任务后可在此查看历史 | [立即运行] |
| 画布空白 | 拖拽引导 | 从左侧拖入步骤开始编排 | 或点击步骤类型直接添加 | — |

### 5.3 加载态

| 场景 | 方案 |
|------|------|
| 列表加载 | 表格骨架屏(5行) |
| 统计卡片加载 | 数字位置 Spin |
