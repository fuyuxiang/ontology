# 数据质量仪表盘 — 设计文档

> 纯展示仪表盘，一屏展示本体数据的整体健康状况。规则由系统自动挂载，前端无配置交互。

## 1. 设计定位

| 维度 | 说明 |
|------|------|
| 页面定位 | 数据质量只读仪表盘，展示本体关联数据资产的健康状况 |
| 核心理念 | 规则自动挂载（代码层）+ 前端纯展示，无需用户手动配置 |
| 用户角色 | 数据治理人员、本体工程师、运维人员 |
| 路由路径 | `/data-integration/quality` |

## 2. 自动规则挂载机制

### 2.1 触发时机

数据资产通过 ObjectBinding 绑定到本体实体时，系统自动为该资产创建质量规则。

### 2.2 推导逻辑

| 实体属性特征 | 自动创建的规则 | 默认参数 |
|-------------|--------------|---------|
| 属性标记 `required: true` | `null_ratio_max` | max: 0.01 (1%) |
| 属性标记为主键（schema_json.primary_key） | `pk_uniqueness` | - |
| 所有绑定资产（通用） | `row_count_min` | min: 1 |
| 所有绑定资产（通用） | `freshness` | max_age_seconds: 86400 (24h) |

### 2.3 实现位置

在 `ObjectBindingService.bind()` 方法中，绑定完成后调用 `QualityRuleService` 自动创建规则。如果规则已存在（同 asset_id + kind + column_name），跳过不重复创建。

## 4. 各区域详细设计

### 4.1 质量得分区（左上）

| 元素 | 规格 |
|------|------|
| 容器 | 高 160px，圆角 16px，背景带微渐变 |
| 环形进度条 | 直径 100px，线宽 10px，颜色渐变 |
| 中心分数 | 36px/700，颜色同进度条 |
| 颜色规则 | ≥80 绿色(#10b981)，60-79 橙色(#f59e0b)，<60 红色(#ef4444) |
| 统计 pill 行 | 分数下方 16px，四个指标水平排列，每个带颜色圆点 |

**得分计算方式：**
- score = (通过规则数 / 总规则数) × 100
- 权重调整: failure 的规则计为 -2 分（而非 -1），使严重问题更突出

### 4.2 实体质量记分卡（左侧主体）

| 元素 | 规格 |
|------|------|
| 容器 | 圆角 12px，内边距 16px |
| 标题行 | "实体健康度" 14px/600 + 实体总数 |
| 每行高度 | 52px |
| 排序 | 得分升序（最差的在最上面） |

**每行结构：**

```
┌──────────────────────────────────────────────────────────────────┐
│ [实体名 14px/500]  [████████░░ 条形图 宽200px]  [92分]  [●●●●○] │
└──────────────────────────────────────────────────────────────────┘
```

- 实体名: 14px/500，最大宽度 140px，超出省略
- 条形图: 高 8px，圆角 4px，背景 neutral-100，填充色根据分数渐变
- 得分数字: 14px/700，颜色同分数规则
- 维度色块: 5 个直径 8px 的圆点，代表:
  - 空值率 (null_ratio_max)
  - 唯一性 (pk_uniqueness)
  - 新鲜度 (freshness)
  - 行数 (row_count_min)
  - Schema (schema_stable)
- 圆点颜色: 绿=通过，橙=告警，红=失败，灰=未检测
- hover: 行背景变为 neutral-50，tooltip 显示各维度具体值

**数据来源：**
- 通过 ObjectBinding 找到有数据资产绑定的实体
- 聚合该实体所有绑定资产的质量规则评估结果
- 无绑定资产的实体不显示

### 4.3 质量趋势图（右上）

| 元素 | 规格 |
|------|------|
| 容器 | 高 200px，圆角 12px |
| 图表类型 | 面积折线图 |
| X 轴 | 近 30 天日期 |
| Y 轴 | 0-100 质量得分 |
| 折线 | 主色 semantic-500，2px |
| 面积填充 | semantic-50，透明度 0.3 |
| 底部叠加 | 告警数(橙色柱)和失败数(红色柱)的迷你柱状图 |

**数据来源：**
- 每天定时评估后记录当日总体得分
- 取 HealthStatus 表按天聚合

### 4.4 最近异常事件流（右下）

| 元素 | 规格 |
|------|------|
| 容器 | flex:1（撑满右侧剩余高度），圆角 12px，内部可滚动 |
| 标题 | "最近异常" 14px/600 |
| 事件条数 | 最近 20 条（warning + failure） |
| 排序 | 时间倒序，最新在上 |

**每条事件结构：**

```
┌─────────────────────────────────────────────┐
│  ⚠  2小时前                                  │
│  基站实体 · device_number 列                  │
│  空值率 12%，阈值 1%                          │
└─────────────────────────────────────────────┘
```

- 状态图标: ⚠(橙) 或 ✗(红)，12px
- 时间: 相对时间，12px，neutral-400
- 实体名 + 列名: 13px/500
- 描述: 12px/400，说明实际值和阈值
- 事件间距: 12px，分割线 1px neutral-100
- 无异常时: 显示 "✓ 所有检查通过" 居中，绿色

```
POST /api/v1/data-plane/quality/evaluate-all
```

触发所有 enabled 规则的评估，返回评估汇总。

## 6. 前端行为

| 行为 | 说明 |
|------|------|
| 页面加载 | 调用 `/quality/dashboard` 渲染全部 |
| 自动刷新 | 每 60 秒轮询一次 |
| "立即检测" | 调用 evaluate-all，按钮 loading 态，完成后刷新仪表盘 |
| 无数据 | 显示空状态："尚无已绑定数据资产的实体" |
| 加载态 | 各区域独立 skeleton |
| 交互 | 仅 hover tooltip（实体行 hover 显示维度详情），无任何编辑/配置 |

## 7. 不做的事

- 不做规则创建/编辑/删除 UI（自动挂载）
- 不做告警配置 UI
- 不做资产选择器
- 不做规则启停开关
- 不做导出功能
- 不做历史详情页（一屏展示完毕）

## 8. 响应式

| 断点 | 布局 |
|------|------|
| ≥1440px | 左 65% + 右 35%，一屏 |
| 1280-1439px | 左 60% + 右 40%，趋势图高度缩小 |
| <1280px | 单列布局，得分区 → 记分卡 → 趋势图 → 异常事件（纵向堆叠） |
