# 企业级 UI 专业化整改设计

> 目标：把当前前端整改为对标"企业版"的专业观感，核心是双列侧边栏多级导航、Ant Design 主题与自研 token 打通、以及新建欢迎首页与核心页面精修。

## 背景与诊断

当前前端技术栈为 Vue 3 + Ant Design Vue 4 + Vite，已有较完整的设计底子：`tokens.css` 定义了语义色板/字号/间距/阴影/圆角，`docs/ui-design/00-design-system.md` 有 293 行设计规范。但实际观感"不够企业级"，根因有三：

1. **导航层次被压平**：`AppSidebar.vue` 现为"灰色假分组标签 + 一级平铺菜单"，分组标签不可点、不可收起，仅一处（对象建模）有二级菜单。多级信息架构没有被表达出来，看着像一堆按钮平铺。
2. **两套皮肤打架**：项目未配置 antd 全局主题，antd 组件仍用其默认蓝（#1677ff）、默认圆角与控件尺寸，和自研 token（语义蓝 #4c6ef5 等）对不上，同页面内组件观感割裂。
3. **缺少落地首页**：`/` 直接重定向到系统看板，没有"欢迎横幅 + 快速入口 + 推荐 + 最近变更"式的专业首页。

整改分三层：全局地基（骨架 + 主题）→ 首页新建 → 核心页面精修。**不改业务逻辑，只改视觉、布局与导航结构。**

## 一、整体骨架：双列侧边栏

改造后外壳分四区：

```
┌────┬──────────┬─────────────────────────────┐
│图标│ 二级菜单栏 │  顶栏（面包屑 + 空间选择 + 用户）    │
│ 栏 │          ├─────────────────────────────┤
│ ▣  │ •对象建模 │                             │
│ □  │  手动构建 │        内容区                │
│ □  │  模板构建 │     （各业务页面）             │
│ □  │ •本体管理 │                             │
│ □  │ •本体服务 │                             │
└────┴──────────┴─────────────────────────────┘
 一级   二/三级              主内容
```

- **图标栏（最左，约 64px）**：竖排 5 个一级模块，每个"图标 + 下方小字"。顶部 Logo，底部折叠按钮。当前模块高亮。
- **二级菜单栏（约 200px）**：显示当前一级模块下的二级/三级菜单，顶部显示模块名。支持三级展开（对象建模 → 手动/模板/文档/资产构建）。
- **顶栏**：保留现有面包屑、空间选择、用户信息。
- **内容区**：各业务页面，结构基本不动。

### 一级模块划分（沿用现有信息架构）

| 一级模块 | 二级 / 三级菜单 |
|---------|----------------|
| 工作台 | 欢迎首页（默认）、业务总览、系统看板 |
| 数据集成 | 数据接入、资产目录 |
| 本体中心 | 对象建模（手动构建/模板构建/文档构建/资产构建）、逻辑建模、行动建模、本体管理、数据映射、本体发布、本体服务 |
| 智能体中心 | 智能体管理、技能管理、流程编排 |
| 运维与安全 | 日志与审计、权限管理、系统配置 |

### 交互细节

- 点图标栏切换模块，二级栏内容随之切换，并自动跳转到该模块默认页。
- 二级栏可整体折叠（只留图标栏），给内容区更多空间。
- 路由变化时，根据当前 `route.path` 反推并高亮对应的一级模块 + 二级菜单项（含三级）。
- 折叠状态用 localStorage 持久化。

### 涉及文件

- `App.vue`：外壳布局由"侧边栏 + 主区"改为"图标栏 + 二级栏 + 主区"。
- `components/common/AppSidebar.vue`：拆分/改造为图标栏 + 二级栏两个组件（如 `AppRailNav.vue` + `AppSubNav.vue`），导航数据结构升级为支持三级的树。
- `AppTopbar.vue`：基本保留，必要时适配新布局的宽度。
- 新增导航配置：把一级/二级/三级菜单树集中定义，供图标栏、二级栏、面包屑复用。

## 二、视觉地基：Ant Design 主题打通

在 `App.vue` 外层包 antd `ConfigProvider`，将 antd 主题令牌接到自研 token，一处配置全站生效：

| antd 令牌 | 接入的 token | 值 |
|-----------|-------------|-----|
| colorPrimary | `--semantic-600` | #4c6ef5 |
| borderRadius | `--radius-md` | 8px |
| fontFamily | `--font-sans` | Inter / Noto Sans SC … |
| colorSuccess / Warning / Error | `--status-*` | 对应状态色 |
| controlHeight / fontSize / lineHeight | 对齐 token 阶梯 | — |

补齐全局细节：

- **表格**：统一表头背景、行高、hover 底色、边框色，观感更紧凑清晰。
- **卡片**：统一 `--shadow-card` + `--radius-lg`，清理页面内零散硬编码阴影。
- **页面容器**：统一内容区 padding、区块间距用 `--space-*`，替换 `margin-top: 16px` 之类硬编码。

**风险**：主题打通后所有 antd 组件颜色/圆角同步变化，绝大多数变好；个别之前靠 antd 默认样式凑合的页面可能需微调，由第三阶段精修覆盖。

## 三、欢迎首页 + 核心页面精修

### A. 欢迎首页（对标图片）

在"工作台"下新建欢迎首页，作为登录后默认落地页，`/` 跳转到此而非系统看板。结构：

- **欢迎横幅**：渐变背景 + "欢迎来到本体平台" + 产品定位一句话 + "开始本体构建"主按钮。
- **快速入口**：高频动作卡片（构建本体、接入数据、创建智能体等）。
- **推荐本体**：卡片网格，无数据时优雅空态。
- **最近变更**：右侧列表，显示最近编辑的本体及时间。

数据尽量接现有接口；接口暂缺处用合理空态，不编造假数据。

### B. 核心页面精修

地基完成后大部分页面自动变好，精修只针对门面页面做结构与排版打磨：

- **第一批（门面）**：欢迎首页、系统看板、业务总览。
- **第二批（核心价值）**：本体管理（对象与关系）、本体图谱、本体发布。
- **第三批（高频）**：数据接入、资产目录、本体服务、智能体管理。

精修内容：统一页面标题区、间距节奏对齐 token、卡片/表格/表单套统一样式、去硬编码、留白透气。**不改业务逻辑。**

### C. 分批交付

1. **阶段一**：地基（antd 主题打通 + 全局细节）+ 骨架（双列侧边栏）+ 欢迎首页 —— 做完即见质变，用户验收。
2. **阶段二**：第一批门面页面精修 —— 验收。
3. **阶段三**：第二批核心价值页面精修 —— 验收。
4. **阶段四**：第三批高频页面精修 —— 验收。

## 验证方式

- 每阶段完成后运行前端构建（`npm run build`，含 vue-tsc 类型检查）确保无类型/编译错误。
- 手动核对导航切换、三级菜单展开/高亮、折叠持久化、路由反推。
- 核对 antd 组件（按钮/表格/表单/弹窗/下拉）颜色圆角是否统一到 token。
- 响应式适配 1920×1080 / 1440×900 / 1280×720 三档分辨率。

## 非目标（YAGNI）

- 不改后端接口与业务逻辑。
- 不做深色模式的额外适配（沿用现有 token 已有的深色定义，不专门优化）。
- 不重构 98 个页面全部，只精修门面与高频页面。
- 不引入新的 UI 框架或图表库。
